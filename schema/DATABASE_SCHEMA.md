# CargoVision AI — Database Schema
**Database Name:** `cargoxai` \
**System Type:** NoSQL (MongoDB)

CargoVision AI uses a flexible NoSQL schema to rapidly ingest deep-learning inference logs in real time. We use two primary collections to track both AI performance and Inspector actions.

---

## 1. Collection: `devloper` (Scans History & Inference Logs)
This collection stores the core analytical telemetry generated by the Model Ensemble (`YOLO` + `Autoencoder`) after an X-Ray is processed.

**JSON Document Schema (`ScanRecord`)**:
```json
{
  "_id":             "ObjectId('...')",        // Internal MongoDB UUID
  "id":              "SHP-1775364122.048",     // Unique Shipment/Scan ID
  "timestamp":       "2026-04-05T10:12:02.12", // ISO-8601 Timestamp of Scan
  "filename":        "xray_crate_004.jpg",     // Original file uploaded
  "declared_type":   "electronics",            // Category from the cargo manifest
  "risk_score":      85.4,                     // Final Ensemble Risk Confidence Score (0-100)
  "risk_level":      "HIGH",                   // Categorical Tier: 'HIGH', 'MEDIUM', 'LOW'
  "total_objects":   4,                        // Count of distinct items identified
  "mismatches":      1,                        // Count of Misdeclaration violations found
  "models_used":     4,                        // Number of ensemble branches contributing
  "model_breakdown": [                         // Individual module telemetry
    { "model": "YOLO-1",              "score": 42.1 },
    { "model": "YOLO-2",              "score": 85.0 },
    { "model": "Autoencoder (Keras)", "score": 90.2 }
  ]
}
```
**Why this design?**
By embedding the `model_breakdown` directly into the document instead of using SQL relational joins, the dashboard's Analytics Engine can retrieve perfectly formatted historical data in a single instantaneous read operation.

---

## 2. Collection: `feedback` (Inspector Ratings & Feedback)
This collection tracks the interactive validations and overrides completed by the Security Inspectors on the dashboard. It serves as our "Ground Truth" annotation pipeline for future re-training.

**JSON Document Schema (`FeedbackRecord`)**:
```json
{
  "_id":         "ObjectId('...')",        // Internal MongoDB UUID
  "shipment_id": "SHP-1775364122.048",     // Links back to the original ScanRecord above
  "rating":      4,                        // Inspector's star rating (1-5) on the AI's accuracy
  "text":        "Good detection but missed small liquid bottle.", // Textual comment or override
  "timestamp":   "2026-04-05T10:14:15.00"  // ISO-8601 Timestamp of submission
}
```
**Why this design?**
Separating feedback from the original scan record means the inference process is never blocked while waiting for human input. This loosely coupled schema allows us to easily match a shipment's AI prediction (`devloper` collection) with actual Human validation (`feedback` collection) during our Machine Learning MLOps pipeline.
